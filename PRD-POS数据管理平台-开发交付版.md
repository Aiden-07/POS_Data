# PRD-POS数据管理平台-开发交付版

## 1. 概要

POS数据管理平台用于统一完成门店POS数据的收取、质量检查、标准化台账汇总、数据分析、权限配置、字段配置和系统日志追踪。

本PRD基于当前项目页面和交互设计整理，目标是让开发、测试、产品可以按模块拆分需求、实现接口、设计数据表并完成验收。

## 2. 干系人

| 角色 | 负责人 | 说明 |
| --- | --- | --- |
| 产品负责人 | 待定 | 负责业务范围、字段标准、验收口径确认 |
| 前端开发 | 待定 | 负责页面、交互、状态管理、导出与本地校验 |
| 后端开发 | 待定 | 负责账号权限、文件处理、质检规则、台账、日志接口 |
| 测试负责人 | 待定 | 负责功能、权限、导出、日志和兼容性测试 |
| 业务管理员 | POS管理团队 | 负责标准字段、角色权限、异常处理规则维护 |

## 3. 项目背景

当前POS数据来自多个门店、经销商、营业所和区域，数据格式不统一、字段命名不一致、异常处理过程分散，导致后续台账汇总和业务分析成本高。

平台需要把“文件收取 -> 质量检查 -> 标准化台账 -> 权限管理 -> 操作留痕”串成一个完整闭环。

现有项目已经包含静态前端原型，核心页面包括：首页概览、文件收取、质量检查、台账与汇总、数据分析、系统设置、AI数据咨询助手。下一步需要补齐后端服务、真实数据、权限生效和可交付级交互。

## 4. 目标

### 4.1 产品目标

1. 统一收取各门店、经销商、营业所上报的POS文件。
2. 按标准字段对原始POS表做质量检查，识别字段缺失、编码异常、重复、金额异常等问题。
3. 生成标准化POS明细台账，并支持筛选、表头配置、分组、导出。
4. 通过角色管理控制功能权限和数据权限，确保用户只能查看授权范围内的数据。
5. 记录用户登录、退出、查看、编辑、修改、驳回、导出等操作，满足审计追溯。
6. 提供AI数据咨询入口，辅助用户查询门店POS数据并生成结果。

### 4.2 成功指标

| 指标 | 目标 |
| --- | --- |
| 文件收取完整率 | 月度文件收取率达到95%以上 |
| 质检自动识别率 | 原始POS异常识别覆盖核心标准字段 |
| 台账生成效率 | 质检通过后可自动进入标准化台账 |
| 权限准确率 | 不同角色访问数据范围与配置一致 |
| 操作留痕完整率 | 关键操作100%进入系统日志 |
| 导出可用性 | 台账、系统日志可导出当前筛选结果 |

## 5. 目标用户

### 5.1 目标用户

| 用户类型 | 工作任务 | 核心诉求 |
| --- | --- | --- |
| POS担当 | 跟进文件收取、处理异常、生成台账 | 快速定位缺失和异常文件 |
| 区域经理 | 查看所辖区域数据进度和质量 | 只能查看自己区域内数据 |
| 营业所负责人 | 管理营业所下经销商和门店数据 | 及时发现门店异常和上报问题 |
| 系统管理员 | 配置用户、角色、字段、日志 | 权限清晰、配置可追踪 |
| 审计/管理人员 | 查看操作记录和数据处理过程 | 操作可追溯，责任清楚 |

### 5.2 数据范围层级

平台数据权限按以下组织层级控制：

```text
区域 -> 营业所 -> 经销商 -> 门店 -> POS明细
```

## 6. 价值主张

1. 对业务人员：减少手工检查文件和字段的时间。
2. 对管理人员：实时了解各区域文件收取、质检和台账进度。
3. 对系统管理员：通过角色统一配置功能权限和数据权限，避免用户单独配置冲突。
4. 对审计人员：所有关键操作有日志，可查看操作前后变化。
5. 对开发团队：字段配置作为标准字段源，质量检查和台账表头共用同一套配置。

### 7.2 全局布局与通用交互

#### 7.2.1 左侧目录

功能要求：

1. 左侧显示系统Logo和一级菜单。
2. 支持折叠和展开。
3. 系统设置支持二级目录：用户管理、角色管理、字段配置、系统日志。
4. 当前页面菜单需要高亮。
5. 数据分析页面当前为待设计页面，仍需在目录中展示。

验收标准：

1. 点击一级菜单后路由切换到对应hash页面。
2. 点击系统设置后展开二级菜单。
3. 折叠目录后保留图标入口，不影响当前页面内容。

#### 7.2.2 顶部栏

功能要求：

1. 展示系统标题：POS数据管理平台。
2. 非“台账与汇总”和非“系统设置”页面展示全局年份、月份选择。
3. “台账与汇总”和“系统设置”下所有页面隐藏顶部全局时间组件。
4. 不展示站内通知入口。
5. 支持语言切换入口，当前原型显示CN。
6. 用户菜单展示用户头像、用户名、角色，点击后支持修改密码、退出登录。

验收标准：

1. 顶部时间组件在首页、文件收取、质量检查展示。
2. 顶部时间组件在台账与汇总、用户管理、角色管理、字段配置、系统日志隐藏。
3. 顶部栏不得出现站内通知按钮、未读角标或通知面板。
4. 点击用户菜单显示下拉菜单。

#### 7.2.3 AI数据咨询助手

功能要求：

1. 页面右下角显示悬浮按钮。
2. 悬浮按钮支持拖拽，并记住上次位置。
3. 默认位置为右下角。
4. 点击悬浮按钮打开数据咨询窗口。
5. 咨询窗口支持历史会话、新建会话、关闭、全屏。
6. 输入年月和门店名称后，生成对应门店POS数据结果。
7. 支持预览结果并导出Excel。

验收标准：

1. 拖拽悬浮按钮后刷新页面，按钮仍在用户放置位置。
2. 点击发送后生成一条用户消息和一条助手回复。
3. 新建会话后历史会话列表增加记录。

关键字支持：

1. 经销商名称。
2. 门店名称。
3. 门店编码。

2. 点击区域后，下方平铺该区域的营业所。

5. 支持查询和重置。

验收标准：

1. 查询后表格数据按筛选条件刷新。
2. 重置后恢复默认年份、月份、组织和空关键字。
3. 所属组织不是下拉框，必须为平铺按钮。

1. 年月。
2. ACC。
3. 经销商名称。
4. 门店编码。
5. 门店名称。
6. 产品编码。
7. 产品名称。
8. 销售数量。
9. 销售金额。

3. 用户可以勾选或取消勾选字段。
4. 至少保留一个字段。
5. 勾选变化后表格表头和数据列即时更新。

#### 7.7.4 分组功能

功能：

1. 点击“分组”按钮打开分组面板。
2. 支持按区域、营业所、经销商单选分组。

4. 选择分组后明细按所选字段分组展示。
5. 分组行支持折叠和展开。

#### 7.7.5 导出功能

功能：

1. 点击“导出”按钮导出当前筛选、当前表头、当前分组状态下的数据。
2. 导出格式优先CSV，后续可扩展Excel。

功能：

1. 表格底部显示当前单据数量。
2. 数量需要随筛选条件、权限范围、分组状态变化而变化。

验收标准：

1. 表头勾选字段后，导出字段与页面字段一致。
2. 分组展开和折叠不影响导出明细。
3. 当前数量与实际明细数据条数一致。

5. 鼠标hover“?”时，在弹窗可视区域内展示数据权限说明浮层。

数据权限策略：

| 策略 | 生效规则 |
| --- | --- |
| 全部数据 | 可以查看全部区域、营业所、经销商、门店和POS明细 |
| 按区域 | 只能查看所选区域下的数据 |
| 按营业所 | 只能查看所选营业所及下属经销商、门店数据 |
| 按经销商 | 只能查看指定经销商及门店数据 |
| 仅本人相关数据 | 只能查看自己创建、处理、上传或被分配的数据 |

功能权限：

按模块配置查看、新增、编辑、审核、导出、删除等权限。

模块范围：

1. 首页概览。
2. 文件收取。
3. 质量检查。
4. 台账与汇总。
5. 数据分析。
6. 系统设置。

验收标准：

1. 保存角色后，角色列表立即更新。
2. 角色数据权限显示策略和范围。
3. 删除角色前需要校验是否有关联用户；有关联用户时提示风险。
4. 角色权限变更后，关联用户立即继承新权限。
5. 角色新增、编辑、删除写入系统日志。

区域、营业所字段来自组织架构，不受字段配置控制。

#### 7.11.2 页面结构

2. 展示字段配置对质量检查和台账与汇总的影响说明。
3. 不展示“质检字段”数量标签。
4. 不展示“台账表头”数量标签。

## 8. 权限模型

### 8.1 功能权限

功能权限控制用户能否访问菜单和执行页面操作。

示例：

| 模块 | 权限点 |
| --- | --- |
| 首页概览 | 查看 |
| 文件收取 | 查看、新增、编辑、导出 |
| 质量检查 | 查看、编辑、审核、导出 |
| 台账与汇总 | 查看、导出 |
| 数据分析 | 查看、导出 |
| 系统设置 | 查看、新增、编辑、删除 |

### 8.2 数据权限

数据权限只在角色中配置，用户继承角色权限。

数据过滤规则：

1. 前端根据当前用户权限隐藏无权菜单和操作按钮。
2. 后端必须在接口层做二次校验。
3. 数据列表、统计卡、导出结果都必须应用数据权限。
4. 不能只依赖前端隐藏。

### 8.3 示例场景

用户Aiden-HB属于“华北区域经理”角色。

角色配置：

1. 功能权限：文件收取、质量检查、台账与汇总、数据分析、导出。
2. 数据权限策略：按区域。
3. 数据权限范围：华北区域。

系统表现：

1. Aiden-HB只能查看华北区域下的营业所、经销商、门店和POS明细。
2. 台账筛选中只能出现华北区域相关组织。
3. 导出结果只能包含华北区域数据。
4. 系统日志记录该用户在授权范围内的操作。

## 10. 非功能需求

### 10.1 性能

1. 普通列表查询响应时间不超过2秒。
2. 台账明细分页查询响应时间不超过3秒。
3. 导出任务超过5秒时应进入异步任务，并给出下载入口。

### 10.2 安全

1. 密码不得明文存储。
2. 接口必须校验token。
3. 后端必须校验功能权限和数据权限。
4. 删除、停用、驳回等高风险操作需要二次确认。
5. 系统日志不可被普通用户删除。

### 10.3 可用性

1. 所有筛选支持重置。
2. 表格空数据时显示空状态。
3. 操作成功或失败需要提示。
4. 弹窗支持关闭。
5. 长表格支持滚动和表头固定。

### 10.4 兼容性

1. 支持Chrome最新版。
2. 桌面端为主，最低宽度建议不低于1280px。
3. 可适配平板宽度，但复杂表格允许横向滚动。
